# 合同風險管理系統 - 項目完成總結報告

## 項目概述

**項目名稱：** 合同風險管理系統（Contract Risk Management System）

**開發時間：** 2024年

**技術棧：**
- 前端：React 18 + TypeScript + Ant Design

## 系統架構

### 1. 數據庫設計

**核心表結構：**
- `users` - 用戶表
- `projects` - 項目表
- `documents` - 文檔表
- `clauses` - 條款表
- `risk_rules` - 風險規則表
- `risk_detections` - 風險檢測結果表
- `annotations` - 批註表
- `document_versions` - 文檔版本表
- `audit_logs` - 審計日誌表

**關係設計：**
```
users (1) ─── (N) projects
projects (1) ─── (N) documents
documents (1) ─── (N) clauses
clauses (1) ─── (N) risk_detections
clauses (1) ─── (N) annotations
documents (1) ─── (N) document_versions
```

### 2. 後端服務

**服務層：**
- `AuthService` - 認證服務
- `ProjectService` - 項目管理
- `DocumentService` - 文檔管理
- `DocumentParserService` - 文檔解析
- `ClauseService` - 條款管理
- `RiskService` - 風險識別
- `AnnotationService` - 批註管理
- `VersionService` - 版本管理
- `ExportService` - 導出服務

**IPC 通信：**
- 統一的 IPC 頻道管理
- 錯誤處理中間件
- 認證中間件
- 標準化響應格式

### 3. 前端架構

**頁面組件：**
- `LoginPage` - 登錄頁面
- `ProjectListPage` - 項目列表
- `DocumentListPage` - 文檔列表
- `DocumentReviewPage` - 文檔審閱
- `UnreadMentionsPage` - 未讀提醒
- `VersionManagementPage` - 版本管理
- `SettingsPage` - 設置頁面

**功能組件：**
- `AppHeader` - 應用頭部
- `ClauseItem` - 條款項
- `AnnotationPanel` - 批註面板
- `ExportModal` - 導出對話框
- `PrivateRoute` - 路由保護

**狀態管理：**
- `AuthContext` - 認證上下文
- `ProjectContext` - 項目上下文

## 核心功能詳解

### 1. 文檔解析引擎

**支持格式：**
- TXT：按段落分割
- DOCX：使用 mammoth 解析
- PDF：使用 pdf-parse 提取文本

**條款識別：**
- 自動編號識別
- 段落分割
- 層級結構分析

### 2. 風險識別系統

**識別方式：**
- 關鍵詞匹配
- 正則表達式
- 風險規則引擎

**風險等級：**
- 高風險（紅色）
- 中風險（橙色）
- 低風險（藍色）

**風險類別：**
- 無限責任條款
- 單方解約權
- 違約金過高
- 管轄權不利
- 保密期限過長

### 3. 批註系統

**批註類型：**
- comment（評論）
- suggestion（建議）
- question（問題）
- issue（議題）

**@mention 功能：**
- 正則表達式解析
- 高亮顯示
- 未讀提醒
- 實時計數

**批註狀態：**
- active（活躍）
- resolved（已解決）
- deleted（已刪除）

### 4. 版本管理

**版本控制：**
- 自動版本號
- 內容快照
- 版本說明
- 創建者記錄

**版本操作：**
- 保存新版本
- 查看歷史版本
- 比較版本差異
- 回滾到指定版本
- 刪除版本

**差異分析：**
- 逐行比較
- 新增內容識別
- 刪除內容識別
- 修改內容識別

### 5. 導出系統

**導出格式：**
- PDF（pdfkit）
- DOCX（docx）
- 審閱報告（docx）

**導出內容：**
- 文檔標題
- 風險摘要
- 條款內容
- 風險標註
- 批註信息

**審閱報告：**
- 風險統計
- 高風險條款清單
- 待確認問題列表
- 建議修改條款清單

## 部署指南

### 開發環境

```bash
# 安裝依賴
npm install

# 啟動開發服務器
npm run dev

# 編譯項目
npm run build
```

### 生產環境

```bash
# 編譯項目
npm run build

# 打包應用
npm run package

# 生成安裝包
npm run make
```

### 系統要求

**最低配置：**
- 操作系統：Windows 10 / macOS 10.13 / Linux
- 內存：4GB RAM
- 硬盤：500MB 可用空間
- Node.js：18.x LTS

**推薦配置：**
- 操作系統：Windows 11 / macOS 12+ / Linux
- 內存：8GB RAM
- 硬盤：1GB 可用空間

## 項目亮點

### 1. 完整的功能閉環

從文檔上傳、解析、風險識別、批註協作、版本管理到最終導出，形成完整的合同審閱工作流。

### 2. 專業的風險識別

基於規則引擎的風險識別系統，支持自定義規則，準確識別合同風險點。

### 3. 強大的協作功能

批註系統支持多種類型，@mention 功能實現團隊協作，未讀提醒確保信息不遺漏。

### 4. 完善的版本控制

類似 Git 的版本管理系統，支持版本比較、回滾，確保文檔修改可追溯。

### 5. 靈活的導出選項

支持多種導出格式，可自定義導出內容，滿足不同場景需求。
