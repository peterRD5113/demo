# 阶段1：保存版本功能测试指南

## ✅ 已完成的修改

### 1. 导入图标
- 已添加 `SaveOutlined` 图标导入

### 2. 添加函数
- 已添加 `handleSaveVersion` 函数
- 功能：调用后端 API 创建新版本快照
- 位置：在 `handleExport` 函数之后

### 3. 添加按钮
- 已在顶部按钮区域添加"保存版本"按钮
- 位置：在"导出"按钮和"版本管理"按钮之间
- 图标：SaveOutlined
- 类型：default（白色背景）

## 🧪 测试步骤

### 前置条件
1. 确保后端服务正在运行
2. 确保数据库已初始化
3. 已登录系统
4. 已上传并解析了一个文档

### 测试流程

#### 步骤1：启动应用
```bash
cd desktop
npm run dev
```

#### 步骤2：进入文档审阅页面
1. 登录系统
2. 选择一个项目
3. 点击进入文档审阅页面

#### 步骤3：测试保存版本功能
1. **查看按钮**
   - 确认顶部有三个按钮：导出、保存版本、版本管理
   - 确认"保存版本"按钮有保存图标

2. **修改条款内容**
   - 点击任意条款进行编辑
   - 修改一些文字内容
   - 保存修改

3. **点击"保存版本"按钮**
   - 点击"保存版本"按钮
   - 应该看到成功提示："版本保存成功"

4. **验证版本已创建**
   - 点击"版本管理"按钮
   - 应该能看到新创建的版本
   - 版本描述应该是："手動保存 - [当前时间]"

#### 步骤4：测试错误处理
1. **测试未登录情况**（如果可能）
   - 应该不会执行任何操作

2. **测试网络错误**（如果可能）
   - 应该显示错误提示："保存版本失敗"

## ✅ 预期结果

### 成功场景
- ✅ 点击按钮后显示："版本保存成功"
- ✅ 版本管理页面能看到新版本
- ✅ 版本描述格式正确
- ✅ 版本内容是当前工作区的快照

### 失败场景
- ✅ 显示错误提示："保存版本失敗"
- ✅ 控制台输出错误信息

## 📊 后端验证

### 数据库检查
```sql
-- 查看最新创建的版本
SELECT * FROM document_versions 
WHERE document_id = [你的文档ID]
ORDER BY created_at DESC 
LIMIT 1;

-- 查看版本的条款快照
SELECT * FROM version_clause_snapshots 
WHERE version_id = [刚创建的版本ID];
```

### API 日志检查
- 查看后端日志，确认 API 调用成功
- 确认返回状态码为 200
- 确认返回数据包含 version_id

## 📝 注意事项

1. **版本描述格式**
   - 当前使用固定格式："手動保存 - [时间]"
   - 未来可以考虑让用户输入自定义描述

2. **性能考虑**
   - 保存版本会创建所有条款的快照
   - 对于大文档可能需要一些时间
   - 考虑添加加载状态提示

3. **用户体验**
   - 成功提示清晰明确
   - 错误提示有助于排查问题
   - 按钮位置合理，易于访问

## 🐛 可能的问题

### 问题1：按钮不显示
- 检查导入是否正确
- 检查 SaveOutlined 是否已导入

### 问题2：点击无反应
- 检查 handleSaveVersion 函数是否正确绑定
- 检查控制台是否有错误

### 问题3：API 调用失败
- 检查后端服务是否运行
- 检查 token 是否有效
- 检查 documentId 是否正确
