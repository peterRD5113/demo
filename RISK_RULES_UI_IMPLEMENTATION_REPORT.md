## 🔌 API 串接

### 前端 → 後端數據流

```
RiskRulesPage.tsx
    ↓ (調用)
window.electronAPI.risk.*
    ↓ (IPC)
Preload API (index.ts)
    ↓ (IPC Channel)
Risk Handlers (riskHandlers.ts)
    ↓ (調用)
Risk Service (RiskService.ts)
    ↓ (調用)
Risk Repository (RiskRepository.ts)
    ↓ (SQL)
SQLite Database (risk_rules 表)
```

### API 方法對應

| 前端操作 | API 方法 | IPC Channel | 後端方法 |
|---------|---------|-------------|---------|
| 載入規則列表 | `getRules()` | `RISK_CHANNELS.GET_RULES` | `riskService.getAllRules()` |
| 新增規則 | `createRule()` | `RISK_CHANNELS.CREATE_RULE` | `riskService.createRule()` |
| 編輯規則 | `updateRule()` | `RISK_CHANNELS.UPDATE_RULE` | `riskService.updateRule()` |
| 刪除規則 | `deleteRule()` | `RISK_CHANNELS.DELETE_RULE` | `riskService.deleteRule()` |
| 啟用/停用 | `updateRule()` | `RISK_CHANNELS.UPDATE_RULE` | `riskService.updateRule()` |

## 📝 使用說明

### 管理員操作指南

#### 新增風險規則
1. 登入管理員帳號
2. 點擊右上角用戶菜單 → 「風險規則管理」
3. 點擊「新增規則」按鈕
4. 填寫表單：
   - **規則名稱**：簡短描述（例如：付款期限過長）
   - **類別**：選擇適當的類別
   - **關鍵字**：用逗號分隔（例如：付款,支付,結算）
   - **匹配模式**：正則表達式（例如：`(?:收到|簽收).*(?:後|內).*(?:[6-9]\d|\d{3,}).*(?:日|天).*付款`）
   - **風險等級**：選擇高/中/低
   - **建議措辭**：給用戶的修改建議
5. 點擊「確定」保存

#### 編輯風險規則
1. 在規則列表中找到要編輯的規則
2. 點擊「編輯」按鈕
3. 修改表單內容
4. 點擊「確定」保存

#### 啟用/停用規則
1. 在規則列表中找到目標規則
2. 切換「啟用/停用」開關
3. 系統自動保存

#### 刪除規則
1. 在規則列表中找到要刪除的規則
2. 點擊「刪除」按鈕
3. 在確認對話框中點擊「確定」
