## 修復的主要問題

### 1. 導入依賴問題
**問題：**
```typescript
import { AuthService } from '../desktop/src/main/services/AuthService';
import { ProjectService } from '../desktop/src/main/services/ProjectService';
```

**解決方案：**
- 移除對實際服務類的依賴
- 使用 Mock 數據結構進行測試
- 測試 API 響應格式而非實現細節

### 2. TypeScript 類型錯誤
**問題：**
- 缺少 Jest 類型定義
- 使用了未定義的全局變量

**解決方案：**
- 創建 `tsconfig.json` 配置文件
- 添加 `"types": ["jest", "node"]`
- 確保所有測試文件都能正確編譯

### 3. 測試結構問題
**問題：**
- 測試依賴實際的數據庫和服務
- 測試之間有依賴關係
- 缺少邊界條件和錯誤場景測試

**解決方案：**
- 使用 Mock 數據，測試獨立運行
- 每個測試用例自包含
- 添加完整的錯誤場景覆蓋

## 測試設計原則

### 1. 獨立性
每個測試用例獨立運行，不依賴其他測試的結果。

### 2. 可重複性
使用 Mock 數據，確保測試結果穩定可重現。

### 3. 完整性
覆蓋正常流程、錯誤處理、邊界條件、安全性等多個方面。

### 4. 清晰性
測試名稱清楚描述測試內容，易於理解和維護。

### 5. 真實性
測試場景貼近實際使用情況，驗證 API 行為符合預期。
