# UI/UX 設計規範

## 1. 設計原則

### 1.1 核心原則

1. **效率優先**: 減少操作步驟，提高審閱效率
2. **信息清晰**: 風險標註醒目，重要信息突出
3. **專業感**: 符合法務人員的專業需求和審美
4. **易用性**: 降低學習成本，符合使用習慣
5. **一致性**: 統一的交互模式和視覺風格

### 1.2 設計目標

- 首次使用無需培訓即可上手
- 常用操作不超過 3 次點擊
- 關鍵信息 3 秒內可定位
- 視覺疲勞度低，適合長時間使用

## 4. 組件設計

### 4.1 頂部導航欄

**高度**: 60px

**內容**:
- 左側: Logo + 應用名稱
- 中間: 當前文檔名稱 + 版本號
- 右側: 
  - 保存按鈕
  - 導出按鈕
  - 設置按鈕
  - 用戶頭像 + 下拉菜單

**交互**:
- 文檔名稱可點擊，顯示文檔信息彈窗
- 保存按鈕顯示保存狀態（已保存/保存中/未保存）
- 用戶頭像下拉菜單包含：個人設置、修改密碼、登出

### 4.2 左側邊欄 - 目錄面板

**功能**:
- 顯示文檔目錄（多級結構）
- 支持摺疊/展開
- 支持搜索條款
- 支持按風險等級過濾

**設計**:

**視圖模式**:

## 5. 交互設計

### 5.1 快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| Ctrl+S | 保存版本 |
| Ctrl+F | 搜索條款 |
| Ctrl+Z | 撤銷 |
| Ctrl+Y | 重做 |
| Ctrl+/ | 添加批註 |
| Ctrl+E | 導出文檔 |
| Ctrl+P | 打印預覽 |
| Ctrl+[ | 摺疊左側邊欄 |
| Ctrl+] | 摺疊右側邊欄 |
| F2 | 重命名文檔 |
| Esc | 關閉彈窗 |

### 5.2 拖拽操作

- 條款可拖拽調整順序
- 模板可拖拽插入到文檔
- 側邊欄寬度可拖拽調整

### 5.3 右鍵菜單

**條款右鍵菜單**:
- 添加批註
- 插入模板
- 調整風險等級
- @同事確認
- 複製條款
- 刪除條款

**文字選中右鍵菜單**:
- 複製
- 高亮標記
- 添加批註
- 搜索相似條款

### 5.4 懸停提示

- 風險標籤懸停顯示風險詳情
- 用戶頭像懸停顯示用戶信息
- 按鈕懸停顯示功能說明
- 截斷文字懸停顯示完整內容

### 5.5 加載狀態

- 文檔導入：進度條 + 百分比
- 風險匹配：骨架屏
- 導出文件：加載動畫 + 提示文字
- 保存版本：按鈕 loading 狀態

### 5.6 錯誤提示

- 輕量級錯誤：Toast 提示（3秒自動消失）
- 重要錯誤：Modal 彈窗（需要用戶確認）
- 表單驗證錯誤：字段下方紅色提示文字

## 6. 動畫效果

### 6.1 過渡動畫

```css
/* 通用過渡 */
transition: all 0.3s cubic-bezier(0.645, 0.045, 0.355, 1);

/* 快速過渡 */
transition: all 0.2s ease-in-out;

/* 慢速過渡 */
transition: all 0.5s ease-in-out;
```

### 6.2 動畫場景

- 側邊欄摺疊/展開：寬度過渡
- 條款展開/摺疊：高度過渡 + 透明度
- 彈窗打開/關閉：縮放 + 透明度
- 列表項添加/刪除：滑入/滑出
- 按鈕懸停：背景色過渡

### 6.3 性能優化

- 使用 `transform` 和 `opacity` 實現動畫（GPU 加速）
- 避免使用 `width`、`height`、`margin` 等觸發重排的屬性
- 長列表使用虛擬滾動

### 7.1 鍵盤導航

### 7.3 語義化 HTML

- 按鈕使用 button 標籤而非 div
